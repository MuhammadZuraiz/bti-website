# Accessibility Checklist

- Skip-to-content link.
- Semantic header, main, footer and nav landmarks.
- One H1 per page.
- Keyboard-accessible navigation.
- Visible focus states.
- Labels for every form field.
- Error messages linked visually and exposed with `role="alert"`.
- Sufficient colour contrast.
- `aria-live` status for form feedback.
- Descriptive image alt text.
- Reduced-motion support.
- RTL-safe layout.
- Accessible mobile menu button.
- Adequate touch targets.
- No essential text embedded in images.
- Manual keyboard and screen-reader spot checks before launch.
